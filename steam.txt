[h1]Exhume 41[/h1]

Run [b]B41 mods on B42[/b] without modification.

B42 introduced versioned subdirectories (e.g. [i]42/media/...[/i]). Mods built for B41 don't have these directories, so the game skips loading their content entirely. This mod:
[olist]
    [*]patches [i]ZomboidFileSystem.getModVersionDirName()[/i] to fall back to the mod root when neither a versioned subdirectory nor [i]common/[/i] exists — making old mods visible to the B42 loader again.
    [*]makes [i]Registry<>.registerBase()[/i] methods available to Lua, so mods can register their content in the "Base" namespace again, as they did in B41. [url=https://steamcommunity.com/sharedfiles/filedetails/?id=3677147974]ZModUnbork[/url] detects if this function is present and uses it to maintain compatibility with B41 mods that register content in the "Base" namespace.
[/olist]

This mod uses the [b]ZombieBuddy 2.1.0+ mod preload mechanism[/b] to apply its patch early enough — before the game resolves any mod content.

[b]Requirements:[/b]

[h1]FAQ[/h1]
[list]
    [*][b]Do map mods work out of the box with this?[/b]
    No. Binary map format changed in B42, so map mods built for B41 won't work without modification.

    [*][b]Does it work for all mods or just listed ones?[/b]
    It should work for all mods, I just listed some popular ones that I know of and have tested with.

    [*][b]Brita's doen't apear on the modlist[/b]
    Be sure you have [url=https://steamcommunity.com/sharedfiles/filedetails/?id=3619862853]ZombieBuddy[/url] installed and activated. It should show its icon and status lines at top left corner of the main menu screen. Be sure it shows "ZBExhume41" in the list of active mods.
[/list]
